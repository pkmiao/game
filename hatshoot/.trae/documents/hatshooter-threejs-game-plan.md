# 《帽孩射击》Three.js 第三/第一人称射击游戏实施计划

## 一、需求摘要（已确认）

| 项目 | 决策 |
|---|---|
| 形式 | `index.html` 入口的多文件网页项目，**完全离线可玩**，无需联网、无需安装任何环境 |
| 角色模型 | 从网上下载尽量接近《时光之帽》原作的模型（帽孩 + 黑手软党反派），不考虑版权 |
| 玩法 | **清场制**：地图固定数量敌人，全灭后胜利结算，可重开 |
| 视角 | 第三人称过肩视角为主，可按 **V 键切换第一人称** |
| 地图风格 | 《镜之边缘》风格：白天、白色楼顶建筑群 + 红色点缀，干净极简 |
| 武器 | 雨伞（近战）、激光枪、榴弹发射器、药瓶手雷 |
| 操作 | 类《战地》/《PUBG》键位，支持冲刺、蹲、**滑铲** |

## 二、关键技术决策（离线 + 小白友好的核心约束）

1. **使用 three.js 的 UMD 单文件版（r128，经典稳定版）**
   - 下载到本地 `libs/three.min.js`，用普通 `<script src>` 标签加载。
   - 这样**双击 index.html 就能玩**（file:// 协议下 ES 模块会被浏览器拦截，UMD 版则不会），完全离线。
2. **不加载任何外部运行时资源**：纹理全部用 Canvas 程序化生成，音效用 WebAudio 实时合成，保证 file:// 下零阻塞。
3. **角色模型的两种落地路径（按优先级尝试）**：
   - **路径 A（优先）**：通过搜索引擎找到可直接下载的帽孩/黑手党风格 GLB 模型 → 转成 base64 内嵌进 `models/*.js` 数据文件（GLTFLoader 支持 data URI），从而兼容 file:// 双击打开。
   - **路径 B（兜底）**：若找不到可直接下载的模型，用代码拼装卡通低模：帽孩 = 紫色高帽 + 黄色上衣 + 蓝色裤子 + 手持雨伞；黑手党 = 白西装 + 圆顶礼帽 + 胡子 + 红色领带。风格与镜之边缘地图统一。
   - 两条路径的接口完全一致（都是返回一个带骨骼动画替代品的 `THREE.Group` + 程序化待机动画），代码结构不受影响。
4. **提供 `start.bat` 备用**：双击后用系统自带命令起本地服务器并自动打开浏览器（以防用户浏览器对 file:// 有额外限制，以及方便以后扩展）。

## 四、键位设计（类战地/PUBG）

| 按键 | 功能 |
|---|---|
| W / A / S / D | 移动 |
| 鼠标 | 视角 |
| 鼠标左键 | 开火 |
| 鼠标右键（按住） | 瞄准（视野拉近、精度提升） |
| Shift（按住） | 冲刺 |
| 空格 | 跳跃 |
| C | 蹲下；**冲刺中按 C → 滑铲**（约 1 秒，速度加成、镜头下沉） |
| R | 换弹 |
| 1 / 2 | 切换 激光枪 / 榴弹发射器 |
| F | 雨伞近战挥击 |
| G | 投掷药瓶手雷 |
| V | 切换 第三人称 / 第一人称 |
| Esc | 暂停（释放鼠标）/ 打开键位说明 |

## 五、玩法与数值（写入 config.js，可调）

- **玩家**：HP 100；战地式**脱战 5 秒后自动回血**；掉出楼顶 → 扣 20 血并传送回出生点。
- **敌人**：黑手党打手 **12 个**，分布在各楼顶平台；HP 100；AI 三态：巡逻（原地晃悠）→ 发现玩家（距离/视线判定）→ 追击并近身挥拳（前摇 0.5 秒，命中 15 伤害）；不会跳楼，追到楼顶边缘停下。
- **武器**：
  - 激光枪：即时命中（hitscan）+ 光束特效；弹匣 30，射速约 8 发/秒，单发伤害 20。
  - 榴弹发射器：抛物线弹道、落地/命中爆炸；弹匣 4，爆炸半径 4 米，中心伤害 80 向外衰减（有自伤，贴脸轰会伤到自己）。
  - 药瓶手雷：按 G 抛出，落地炸开**紫色毒雾区**，持续 3 秒范围掉血；随身 3 个。
  - 雨伞：F 挥击，面前扇形 2.5 米，伤害 35 + 击退，冷却 0.6 秒。
- **胜负**：HUD 显示「剩余敌人 x/12」；全灭 → 胜利结算（通关用时、剩余血量），死亡 → 失败画面；两者都可按键重开一局。

## 六、地图设计（镜之边缘风格）

- 6~8 栋不同高度的白色大楼组成屋顶跑酷区，楼体为干净的白盒 + 浅灰边框。
- 红色点缀：红色消防门、红色管道、红色集装箱/空调箱（既美观也当掩体）。
- 楼与楼之间有可跳跃的间隙和少量木板桥；远景放一圈更大的剪影楼群。
- 白天光照：平行光 + 半球光，淡蓝天空渐变背景 + 轻雾，画面干净明亮。
- 碰撞：楼体使用 AABB（轴对齐包围盒），自写轻量碰撞（胶囊 vs 盒），不引入物理引擎，保持简单可靠。

## 七、实施步骤（执行顺序）

1. **下载 three.js r128 UMD 版**到 `libs/three.min.js`（联网下载一次，之后永久离线）。
2. **尝试路径 A**：搜索可直连下载的帽孩/黑手党 GLB 模型；成功则下载并转 base64 写入 `models/`；失败则走路径 B 程序化拼模。
3. 搭骨架：`index.html` + `main.js` + `config.js` + `input.js`，渲染出空场景和指针锁定。
4. `world.js`：生成地图、光照、天空、碰撞数据。
5. `player.js` + `cameraRig.js`：移动、冲刺、跳、蹲、**滑铲**、碰撞、掉落重生、双视角切换。
6. `weapons.js` + `effects.js` + `audio.js`：四种武器、弹道/爆炸/光束/粒子、合成音效。
7. `enemies.js`：黑手党模型 + AI + 受击死亡 + 剩余计数。
8. `ui.js`：完整 HUD、开始画面（点击开始并锁定鼠标）、胜利/失败结算、暂停键位表。
9. `start.bat` + `README.txt`。
10. **联调与验收**（见下节）。

## 八、验收方式

1. 本地起临时服务器，用浏览器自动化打开页面：确认控制台无报错、场景正常渲染、能看到帽孩和敌人。
2. 模拟输入做冒烟测试：移动、开枪命中敌人掉血、榴弹爆炸、手雷毒雾、滑铲触发、V 键切换视角。
3. 全灭敌人触发胜利结算；玩家被打死触发失败画面；重开正常。
4. 最后由你**双击 index.html** 实测（这是给小白的主路径），如有浏览器限制则改用 `start.bat`。

## 九、风险与说明

- **模型下载不确定性**：免费 3D 模型站（如 Sketchfab）多数下载需要登录，自动化直连下载不一定成功。若路径 A 失败会自动采用路径 B 的拼模方案，并尽量还原配色与特征（紫帽黄衣的帽孩、白西装礼帽胡子的黑手党）。
- **第一人称视角**：第一人称时隐藏帽孩身体、显示简易武器视图模型（避免做两套手臂动画）。
- **敌人全部为近战挥拳型**（还原黑手党打手风格），首版不做远程敌人。
